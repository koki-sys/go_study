# 🧭 Golang 学習ロードマップ（Java エンジニア向け）

## 1️⃣ 基礎強化フェーズ（2〜3 週間）

**目的:** Go の思想（シンプルさ・明示性）を体に馴染ませる。

### 内容

-   **Go の設計思想理解**
    -   _なぜ構造体なのか？なぜ継承がないのか？_
    -   Java との違いを意識しながら進める。
-   **標準パッケージを軽く触る**
    -   `fmt`, `strings`, `time`, `io`, `net/http`
-   **文法復習＋補強**
    -   Interface, Goroutine, Channel
    -   `defer`, `panic`, `recover` の動き

### 演習

-   Java で作った小ツールを Go で書き直す。
    -   例：ファイル読み込み＋文字数カウント
    -   REST API モックサーバー（`net/http`で簡単に）

### Java 比較メモ

| 項目         | Java          | Go                     |
| ------------ | ------------- | ---------------------- |
| 継承         | クラス階層    | 構造体＋インタフェース |
| 例外         | try-catch     | 戻り値＋`error`型      |
| スレッド     | 重い Thread   | 軽量 Goroutine         |
| ジェネリクス | Java17 で安定 | Go1.18〜最小限実装     |

## 2️⃣ 中級：並行処理・設計（3〜4 週間）

**目的:** Go らしい書き方で、処理を分け・まとめる。

### 内容

-   **並行処理**
    -   `go`キーワードで非同期実行
    -   `sync.WaitGroup`, `channel`の使い方
-   **テストとエラーハンドリング**
    -   `testing`パッケージでユニットテスト
    -   `errors.Is`, `errors.As`で安全に扱う
-   **プロジェクト構成**
    -   `cmd/`, `pkg/`, `internal/`ディレクトリ構造を知る

### 演習

-   CSV を並列でパースして集計する処理
-   HTTP API の非同期呼び出しを Goroutine で最適化
-   `context`を用いたキャンセル・タイムアウト処理

### 理解度チェック

-   goroutine の終了を安全に待つ方法を説明できるか
-   エラーラップ（`fmt.Errorf("%w", err)`）を使えるか
-   テストを`go test`で回し、カバレッジを確認できるか

## 3️⃣ 実践：API・DB・設計パターン（1〜2 ヶ月）

**目的:** 実務とつながる形に持っていく。

### 内容

-   **Web フレームワーク**
    -   軽量：`Gin`, `Echo`
-   **ORM・DB**
    -   `GORM` or `sqlx`（まずは生 SQL も体験）
-   **DI・設定管理**
    -   Java 的 DI を求めすぎず、構造体注入＋初期化関数で対応
-   **API 設計**
    -   Handler 層と UseCase 層を分ける Clean Architecture 的整理

### 演習

-   TODO 管理 API を Gin で実装
-   SQLite/PostgreSQL と接続し CRUD を実装
-   Docker でローカル実行

### 理解度チェック

-   HTTP ハンドラとミドルウェアの関係を説明できる
-   Go における依存の向き（下 → 上ではなく、上 → 下）を理解できる
-   ログと設定を`config.yaml`で管理できる

## 💡 Java エンジニアが Go を業務に活かすヒント

-   **API 設計の発想は共通**。ただし Go は“薄い層”を好む。抽象を増やしすぎない。
-   **構造体＋ interface**で依存を反転させるのが、Go の“DI”。
-   **Go Modules**でのバージョン管理は、Maven/Gradle より軽いが慎重に扱う。
-   Spring の「設定マシマシ」な世界から「コードで表現する設計」への転換。

## 🧩 詰まったときの対処法

-   **原因の切り分けを言語化する**
    -   “どこまで動いてるか”を`fmt.Println`で確認
-   **Go Playground で最小再現**
    -   環境汚染を防げる
-   **質問テンプレ**
    -   「○○ をしたい → △△ を試した → ✕✕ で詰まった」  
        → Go の界隈はこの書き方が非常に好まれる
